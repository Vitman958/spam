# 🛡️ Spam Filter

Приложение для фильтрации спам-сообщений на основе машинного обучения с графическим интерфейсом.

## Описание

Spam Filter — это десктопное приложение, которое использует наивный байесовский классификатор для классификации текстовых сообщений на спам и не спам. Приложение построено с использованием библиотеки `customtkinter` для современного графического интерфейса.

## Установка

1. Клонируйте репозиторий:
```bash
git clone <repository-url>
cd <project-directory>
```

2. Установите зависимости:
```bash
pip install -r requirements.txt
```

3. Скачайте необходимые корпуса NLTK:
```bash
python scripts/download_nltk.py
```

## Использование

### Запуск приложения

```bash
python main.py
```

После запуска откроется графический интерфейс, где вы сможете ввести текст и проверить его на спам.

### Обучение модели

Если у вас есть датасет для обучения:

1. Поместите исходный датасет в папку `data/raw/`

2. Обработайте данные:
```bash
python scripts/process_dataset.py
```

3. Обучите модель:
```bash
python scripts/train_model.py
```

Модель будет сохранена в папке `models/`.

## Как это работает

1. **Предобработка текста**: Текст проходит через конвейер обработки:
   - Приведение к нижнему регистру
   - Замена URL, email, HTML-тегов на специальные токены
   - Удаление пунктуации
   - Токенизация
   - Удаление стоп-слов
   - Лемматизация

2. **Векторизация**: Используется TF-IDF векторизатор с n-граммами (1-2)

3. **Классификация**: Модель Multinomial Naive Bayes предсказывает вероятность того, что сообщение является спамом

4. **Пороговое значение**: Сообщение считается спамом при вероятности > 75%

## Структура данных

Ожидаемый формат входных данных (CSV):
- `v1` или `label` — метка класса ('spam' или 'ham')
- `v2` или `message` — текст сообщения

## Настройка

Параметры можно изменить в файле `src/classifier.py`:
- `ngram_range` — диапазон n-грамм
- `min_df`, `max_df` — параметры частоты терминов
- `threshold` — порог классификации (по умолчанию 0.75)
